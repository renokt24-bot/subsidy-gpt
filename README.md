# subsidy-gpt — 助成金マッチングツール

会社情報を入力すると、**使える可能性のある助成金・補助金をAIが優先度づけして表示する**ブラウザツールです。
インストールやAPIキーは不要。`index.html` を開くだけで動きます。

2種類のデータをまとめて判定します。

1. **収録ルール（`data.js`）** — 厚生労働省の雇用・人材系助成金など、要件を細かくルール化した代表制度
2. **公式の最新データ（jGrants 連携）** — 国・**地方自治体（都道府県・市区町村）**の**募集中の補助金**を
   政府の補助金ポータル [jGrants](https://www.jgrants-portal.go.jp/) の公開APIから自動取得

## 主な機能

| 機能 | 説明 |
| --- | --- |
| 🔎 **法人番号検索** | 会社名から法人情報を検索し、所在地を自動入力（国税庁API設定時は本番データ、既定はデモ用サンプル） |
| 🤖 **AI補助金診断** | 目的一致・採択率・締切・補助額・地域から各制度を0〜100点でスコアリングし、優先度（高/中/低）順に表示 |
| 📊 **採択率表示** | 補助金の代表的な採択率、助成金は「要件を満たせば支給」を明示 |
| ↕️ **並び替え** | AI優先度順／申請期限が近い順／採択率が高い順／補助額が大きい順 |
| ✉️ **申請代行フォーム** | 申請サポートへの問い合わせフォーム（`mailto:` 送信＋ローカル保存） |
| ⚙️ **管理画面** | 独自の補助金データを追加・編集・削除（localStorage保存／JSONエクスポート・インポート） |
| 📱 **スマホ最適化** | スマートフォン表示に対応したレスポンシブUI |

> **設定**: 申請代行の問い合わせ先メールや法人番号APIは [`config.js`](./config.js) で変更できます。

## ローカルでの動かし方

このリポジトリを取得し、`index.html` をブラウザで開くだけです（インストール不要）。

```bash
# もっとも簡単（ファイルをダブルクリックでもOK）
open index.html              # macOS
xdg-open index.html          # Linux
# または簡易サーバーで
python3 -m http.server 8000  # → http://localhost:8000
```

## 画面と使い方

本ツールは4画面で構成されています（詳細は [画面設計書](./docs/画面設計.md)）。

1. **入力フォーム画面** — 会社情報を入力
   - 会社名（任意・**法人番号検索**で所在地を自動入力）
   - 所在地（都道府県）／業種／従業員数／資本金／創業からの年数
   - **やりたいこと**（採用、正社員化、研修、賃上げ、設備投資、IT導入、販路開拓、創業 など）を複数選択
2. 「AIで使える助成金を診断する」を押すと **補助金一覧画面** に切り替わり、**AI診断による優先度順**で表示
   - 各カードに **優先度バッジ・スコア・採択率・申請締切・診断理由** を表示
   - 上部の **並び替え** で表示順を変更（AI優先度／申請期限／採択率／補助額）
   - 各カードの「申請代行に相談」から **申請代行フォーム** へ
3. **申請代行フォーム画面** — 専門スタッフへの相談を送信
4. **管理画面** — 独自の補助金データを追加・編集（上部ナビ「管理画面」から）

## 🤖 AI事業計画書アシスタント（革新機能・プロトタイプ）

「補助金を探す」だけでなく、**採択率を左右する事業計画書づくりまでAIが伴走**する機能です。
対象は「小規模事業者持続化補助金」。数問のヒアリング → **事業計画書ドラフトをAIが生成** →
**審査基準に照らした採点・改善提案（採択率診断）** を行います。

- 画面: [`plan.html`](./plan.html)（上部ナビ「AI計画書」）
- モデル: **Claude Opus 4.8**（`claude-opus-4-8`）。ドラフトはストリーミング生成、採点は構造化出力。
- バックエンド必須: APIキーはブラウザに置けないため、小さなNodeサーバー（[`server/`](./server)）経由で呼び出します。

### 動かし方（ローカル）

```bash
cd server
npm install
export ANTHROPIC_API_KEY="sk-ant-..."   # 自分のAnthropic APIキー
npm start                                # → http://localhost:5179/plan.html
```

`http://localhost:5179/` を開くと、静的サイト本体（診断・管理画面）も同じサーバーから配信されます。
APIキー未設定でも画面は表示されますが、生成・診断はキー設定後に動作します。

> この機能はサーバーが必要なため **GitHub Pages（静的）では動きません**。Pages上の `plan.html` は
> 「ローカルサーバーが必要」と表示します。デプロイする場合はNodeが動くホスト（Render/Fly.io等）に
> `server/` を置いてください。

> ⚠️ プロトタイプの注意：制度知識（審査基準）は公募要領の**要約版**を `server/subsidy-spec.js` に
> 収録しています。実運用では公募要領PDFを Files API + Citations で読み込み、根拠付けを厳密化する想定です。
> 生成された数値・固有名詞は必ずご自身で検証してください。

## 公式データの自動連携（jGrants）

国・地方自治体の募集中補助金は、政府の補助金ポータル **jGrants 公開API** から取得します。

```bash
# 最新データを取得して data/subsidies.json と data/subsidies.js を再生成
node scripts/fetch-jgrants.js
```

- 取得対象は **募集中（acceptance=1）** の補助金。所在都道府県・従業員数・募集期間・目的で自動絞り込み。
- 生成物の `data/subsidies.js` をブラウザが読み込むため、**サーバー不要**で `file://` のまま動きます。
- **自動更新**: [`.github/workflows/update-subsidies.yml`](./.github/workflows/update-subsidies.yml) が
  毎日データを再取得し、変更があれば自動コミットします（手動実行も可）。
- 出典・データ更新日は結果画面に表示されます。

> jGrants APIはCORSヘッダーを返さないため、ブラウザからの直接呼び出しはできません。
> そのため「取得スクリプト → 静的データ生成 → アプリが読込」という構成にしています。

## 制度を追加・編集するには

### 方法A: 管理画面（GUI・コード不要）

上部ナビの **「管理画面」** から、独自の補助金データをフォームで追加・編集・削除できます。
- 登録データはブラウザの localStorage に保存され、診断結果にも即反映されます。
- **エクスポート / インポート**（JSON）でデータの共有・バックアップが可能です。
  エクスポートしたJSONをリポジトリに含めれば、配布データとして管理できます。

### 方法B: コードで収録（`data.js`）

組み込みの制度データは [`data.js`](./data.js) に定義されています。
`SUBSIDIES` 配列にオブジェクトを追加するだけで新しい制度を収録できます。
`SUBSIDY_META` に `purposes`（目的タグ）と `adoptionRate`（採択率）を加えると、AI診断のスコアに反映されます。

```js
{
  id: "example",
  name: "制度名",
  org: "所管",
  category: "カテゴリ",
  amount: "支給額の目安",
  summary: "概要",
  url: "https://...",
  match(c) {
    // c: { prefecture, industry, employees, capital, yearsInBusiness, purposes[] }
    if (!c.purposes.includes("equipment")) return { eligible: false };
    return {
      eligible: true,
      reasons: ["あてはまる理由"],
      cautions: ["確認が必要な条件"],
    };
  },
}
```

中小企業・小規模事業者の判定には `isSME(c)` / `isSmallBusiness(c)` を利用できます。

## 注意

本ツールの判定は **申請の目安** です。各制度の要件・支給額・募集状況は簡略化しています。
実際の支給可否は、必ず厚生労働省・中小企業庁・各自治体・お近くの労働局や商工会議所等の
**公式な最新情報** でご確認ください。
