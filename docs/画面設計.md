# 画面設計書 — 助成金マッチングツール（subsidy-gpt）

## 1. 画面一覧

| ID | 画面名 | 役割 |
| --- | --- | --- |
| S1 | 入力フォーム画面 | 会社情報を入力（法人番号検索を含む）して診断する |
| S2 | 補助金一覧画面 | AI診断で優先度づけした助成金・補助金を一覧表示する |
| S3 | 申請代行フォーム画面 | 申請代行サービスへの問い合わせを送信する |
| S4 | 管理画面 | 独自の補助金データを追加・編集・削除する |

4画面は1つの `index.html` 内のセクションとして実装し、JavaScriptで表示を切り替える（SPA方式）。
全画面共通の上部ナビに「診断（S1）」「管理画面（S4）」へのリンクを配置する。

## 2. 画面遷移図

```
   ナビ:[診断] [管理画面]
        │           └──────────────▶ ┌──────────────────┐
        ▼                            │ S4 管理画面        │
   ┌──────────────────────┐         │ 追加/編集/削除      │
   │ S1 入力フォーム画面     │         │ エクスポート/インポート│
   │ [法人番号検索]          │         └──────────────────┘
   │ [AIで診断する] ─────────┼──▶ 入力チェックOK
   └──────────────────────┘        │
             ▲                      ▼
             │            ┌──────────────────────┐
   [← 条件を変更] ─────────┤ S2 補助金一覧画面（AI診断）│
                          │  優先度バッジ/採択率/締切   │
                          │  並び替え（4種）           │
                          └──────────────────────┘
                              │                  │
              [公式の詳細を見る →]      [申請代行に相談] ──┐
                  別タブで公式サイト                      ▼
                                          ┌──────────────────────┐
                              [← 一覧へ戻る]─┤ S3 申請代行フォーム画面  │
                                          │  mailto: で送信         │
                                          └──────────────────────┘
```

- 入力チェックNG時はS1に留まり、エラーメッセージを表示。
- 「← 条件を変更」でS1へ戻る（入力値は保持）。
- 「申請代行に相談」は対象補助金名をプリフィルしてS3を開く。

## 3. 共通要素

- **ヘッダー（hero）**: タイトル「助成金マッチング」＋サブコピー。全画面共通で上部に表示。
- **フッター（disclaimer）**: 判定は目安である旨の免責文。全画面共通。
- **配色**: ブランドカラー＝青（`#2563eb`）。公式補助金カードは水色アクセント（`#0ea5e9`）。
- **レスポンシブ**: 画面幅560px以下でフォームを1カラム化。

## 4. S1 入力フォーム画面

### ワイヤーフレーム

```
┌────────────────────────────────────────┐
│            助成金マッチング               │  ← hero
│  会社情報を入力すると…すぐにわかります       │
├────────────────────────────────────────┤
│  会社情報を入力                           │
│  ┌─────────────┐ ┌─────────────┐        │
│  │ 所在地(都道府県)│ │ 業種          │        │
│  └─────────────┘ └─────────────┘        │
│  ┌─────────────┐ ┌─────────────┐        │
│  │ 従業員数(人)  │ │ 資本金(万円)   │        │
│  └─────────────┘ └─────────────┘        │
│  ┌─────────────┐                        │
│  │ 創業からの年数 │                        │
│  └─────────────┘                        │
│  やりたいこと（複数選択可）                 │
│  [採用][正社員化][研修][賃上げ][設備投資]    │  ← チップ（チェック式）
│  [IT導入][販路開拓][創業][事業転換][両立支援] │
│                                          │
│  (エラー時) ⚠ すべての会社情報を入力…       │
│  ┌────────────────────────────────────┐ │
│  │        使える助成金を調べる            │ │  ← CTAボタン
│  └────────────────────────────────────┘ │
└────────────────────────────────────────┘
```

### 入力項目定義

| 項目 | コントロール | 必須 | 値 | バリデーション |
| --- | --- | --- | --- | --- |
| 会社名（法人番号検索） | テキスト＋検索ボタン | 任意 | 自由入力 | 検索ヒットを選ぶと所在地を自動入力 |
| 所在地（都道府県） | セレクト | ○ | 47都道府県 | 未選択不可 |
| 業種 | セレクト | ○ | 製造業他／卸売／小売／サービス | 未選択不可 |
| 従業員数 | 数値入力 | ○ | 0以上の整数 | 未入力不可 |
| 資本金（万円） | 数値入力 | ○ | 0以上の整数 | 未入力不可 |
| 創業からの年数 | 数値入力 | ○ | 0以上の整数 | 未入力不可 |
| やりたいこと | チェックチップ | 任意 | 10種から複数 | なし |

> 先頭に **法人番号検索** ブロックを配置。会社名を入力して「検索」すると候補を一覧表示し、
> 選択すると所在地（都道府県）を自動入力する。CTAは「AIで使える助成金を診断する」。

### 「やりたいこと」の選択肢

採用・雇用 / 非正規の正社員化 / 社員研修・人材育成 / 賃上げ / 働き方改革・両立支援 / 設備投資・生産性向上 / ITツール・DX導入 / 販路開拓・新商品開発 / 創業・起業 / 新分野展開・事業転換

### 操作・挙動

- 「AIで使える助成金を診断する」押下 → 必須チェック → OKならS2へ遷移しつつ診断実行。
- NGならエラー文を表示し、最初の未入力項目にフォーカス。

## 5. S2 補助金一覧画面（AI診断結果）

### ワイヤーフレーム

```
┌────────────────────────────────────────┐
│            助成金マッチング               │  ← hero（共通・ナビ付き）
├────────────────────────────────────────┤
│ [← 条件を変更]  N件の候補   並び替え[▼]    │  ← results-bar
│ [AI診断] N件を優先度づけしました（高：M件） │  ← AI診断サマリー
│ 公式補助金の出典: jGrants（更新: 2026/...） │  ← 出典・更新日
│  ┌────────────────────────────────────┐ │
│  │ [優先度 高・87点] [カテゴリ/地域]       │ │  ← 優先度バッジ
│  │ 制度名                               │ │
│  │ 所管／実施機関                        │ │
│  │ ┌支給/補助額┐┌採択率┐┌申請締切┐       │ │  ← 統計（stats）
│  │ 🤖 診断: 目的2件一致／採択率60%／締切… │ │  ← AI診断の理由
│  │ ✓ あてはまる点 ／ ！確認が必要な条件    │ │  ←（ルール系のみ）
│  │ 公式の詳細を見る →   [申請代行に相談]   │ │  ← リンク＋代行ボタン
│  └────────────────────────────────────┘ │
│  …（優先度順にカードが続く）                │
│  ┌──── 申請代行への導線（CTA）────────────┐ │
│  │ 迷ったら専門スタッフがサポート          │ │
│  │ [申請代行サービスに相談する]            │ │
│  └────────────────────────────────────┘ │
└────────────────────────────────────────┘
```

### 表示構成

1. **ヘッダーバー**: 「← 条件を変更」＋件数＋**並び替えセレクト**
2. **AI診断サマリー**: 件数・優先度高の件数・スコアリング根拠の注記
3. **統合カード一覧**（ルール系＋jGrantsを1リストに統合し、選択した順序で並べる）

### 並び替え（4種）

| 値 | 並び順 |
| --- | --- |
| AI優先度順（既定） | 診断スコアの高い順 |
| 申請期限が近い順 | 締切（acceptanceEnd）の近い順（締切なしは末尾） |
| 採択率が高い順 | 採択率の高い順（助成金=要件支給を上位扱い） |
| 補助額が大きい順 | 補助上限額の大きい順 |

### カード項目（統合）

| 項目 | 内容 |
| --- | --- |
| 優先度バッジ | 高（赤）/中（黄）/低（灰）＋スコア。カード左ボーダーも優先度色 |
| バッジ | カテゴリ／地域／従業員規模／「独自登録」 |
| 統計（stats） | 支給・補助額／採択率（または「要件を満たせば支給」）／申請締切 |
| AI診断の理由 | スコアの主要因を1行で表示 |
| あてはまる点／確認が必要な条件 | ルール系・独自登録のみ |
| アクション | 公式リンク＋「申請代行に相談」ボタン |

### 状態（ステート）

| 状態 | 表示 |
| --- | --- |
| 候補あり | AI診断サマリー＋優先度順カード群＋申請代行CTA |
| 候補なし | 「条件に一致する制度はありませんでした」＋再検索の案内文 |

## 6. S3 申請代行フォーム画面

```
┌────────────────────────────────────────┐
│ [← 一覧へ戻る]                            │
│ 申請代行サービスへのお問い合わせ            │
│ ┌会社名*┐ ┌ご担当者名*┐                  │
│ ┌メール*┐ ┌電話┐                         │
│ ┌相談したい補助金（プリフィル可）┐          │
│ ┌相談内容*（テキストエリア）┐               │
│ (エラー/送信完了メッセージ)                 │
│ [この内容で問い合わせる]                    │
└────────────────────────────────────────┘
```

- 必須: 会社名／ご担当者名／メール／相談内容。メールは形式チェック。
- 送信時: `localStorage` に記録し、`config.js` の宛先へ `mailto:` でメール作成。
- 「申請代行に相談」から遷移した場合、対象の補助金名をプリフィル。

## 7. S4 管理画面

```
┌────────────────────────────────────────┐
│ 管理画面 — 補助金データの追加・編集         │
│ ┌登録済みデータ一覧（表）─────────────┐    │
│ │ 制度名 | 所管 | カテゴリ | 採択率 | 操作│    │
│ │ …                         [編集][削除]│    │
│ └──────────────────────────────────┘    │
│ [エクスポート] [インポート] [全削除]        │
│ ── 補助金を追加 / 編集 ──                   │
│ ┌制度名*┐┌所管┐┌カテゴリ┐┌金額┐         │
│ ┌採択率┐┌URL┐┌従業員下限┐┌上限┐        │
│ ┌概要（テキストエリア）┐                    │
│ 対象の目的[チップ複数]                       │
│ 対象都道府県[マルチセレクト]                  │
│ [この内容で追加] [クリア]                    │
│ ▸ 内蔵の制度（12件・読み取り専用）            │
└────────────────────────────────────────┘
```

- 一覧は localStorage の独自データを表示。各行に「編集／削除」。
- 「編集」でフォームに値を読み込み、保存で上書き更新。
- エクスポート＝JSONダウンロード、インポート＝JSON読み込み、全削除＝一括クリア。
- 内蔵制度は読み取り専用で参照表示（編集は `data.js`）。

## 8. 文言・トーン

- 専門用語にはふりがな的な補足を添え、非エンジニア・非専門家にも分かる平易な日本語。
- 断定を避け「使える可能性」「目安」「参考値」と表現。最終確認は公式情報へ誘導。
