# 設計書 — 助成金マッチングツール（subsidy-gpt）

## 1. 目的・概要

会社情報（所在地・業種・規模・やりたいこと）を入力すると、**利用できる可能性のある助成金・補助金を即座に一覧表示**するツール。中小企業・小規模事業者が「自社で使える制度」を素早く把握することを目的とする。

判定対象は次の2系統。

| 系統 | 内容 | データ源 |
| --- | --- | --- |
| ルールベース助成金 | 厚労省の雇用・人材系助成金など、要件を細かくルール化した代表制度 | `data.js`（手動メンテ） |
| 公式・募集中補助金 | 国・地方自治体の募集中補助金 | jGrants公開API（自動取得） |

## 2. 技術選定

- **フロントエンド**: 素のHTML / CSS / JavaScript（フレームワーク・ビルド不要）
- **理由**: インストール不要・`file://` で直接開けることを最優先。非エンジニアでもダブルクリックで動く。
- **データ取得**: Node.js（標準 `fetch` のみ。外部依存パッケージなし）
- **自動更新**: GitHub Actions（スケジュール実行）
- **公開（任意）**: GitHub Pages

## 4.1 主な機能と実装方針

| 機能 | 実装方針 |
| --- | --- |
| 法人番号検索 | `corporate.js`。`config.js` にAPI（プロキシ）を設定すれば本番、未設定はデモ用サンプル。国税庁APIはCORS非対応のため同一オリジンのプロキシを想定。取得できるのは商号・所在地のみ（従業員数等は手入力）。 |
| AI補助金診断 | `app.js` の `diagnose()` による透明なスコアリング（0〜100点）。LLM不要で `file://` 動作。将来LLMに差し替え可能。 |
| 採択率表示 | `data.js` の `SUBSIDY_META` に代表値を保持。助成金は `nonCompetitive`（要件を満たせば支給）として区別。 |
| 並び替え | 統合候補配列を `sortCandidates()` でキー別ソート（AI優先度／締切／採択率／補助額）。 |
| 申請代行フォーム | `mailto:` でメール作成＋ `localStorage` に問い合わせ記録。宛先は `config.js`。 |
| 管理画面 | 独自データを `localStorage`（`subsidy-admin-custom-v1`）に保存し、診断時に組み込みデータと統合。JSONエクスポート/インポート対応。 |
| スマホ最適化 | `styles.css` のメディアクエリ（640px以下）でレイアウト・タップ領域・文字サイズを調整。 |

## 4.2 AI診断のスコアリング

各候補に対し以下を加点し、0〜100点に正規化して優先度（高≥70／中≥48／低）を決定する。

| 観点 | 配点 | 内容 |
| --- | --- | --- |
| 目的一致 | 最大35 | ユーザーが選んだ「やりたいこと」と制度の目的タグの一致数（×18、上限35） |
| 採択率・支給型 | 最大25 | 採択率に比例。助成金（`nonCompetitive`）は高めの固定値、不明は中立値 |
| 締切の近さ | 最大20 | `acceptanceEnd` までの日数が近いほど高い（締切なしは中立値） |
| 補助額 | 最大12 | 補助上限額が大きいほど高い |
| 地域密着 | 最大8 | 自治体（都道府県指定）の制度を全国制度より優先 |

## 5. データモデル

### 5.1 会社情報（入力 / company）

| フィールド | 型 | 説明 |
| --- | --- | --- |
| `prefecture` | string | 所在都道府県 |
| `industry` | string | 業種ID（`manufacturing` / `wholesale` / `retail` / `service`） |
| `employees` | number | 従業員数（人） |
| `capital` | number | 資本金（万円） |
| `yearsInBusiness` | number | 創業からの年数 |
| `purposes` | string[] | やりたいことのID配列（複数選択可） |

### 5.2 ルールベース助成金（data.js / SUBSIDIES[]）

| フィールド | 説明 |
| --- | --- |
| `id` / `name` / `org` / `category` | 識別子・制度名・所管・カテゴリ |
| `amount` / `summary` / `url` | 支給額目安・概要・公式URL |
| `match(company)` | 判定関数。`{ eligible, reasons[], cautions[] }` を返す |

補助関数 `isSME(company)` / `isSmallBusiness(company)` で中小企業・小規模事業者を業種別に判定。

### 5.3 公式補助金（data/subsidies.json / subsidies[]）

| フィールド | 説明 |
| --- | --- |
| `id` / `title` / `institution` | jGrants ID・制度名・実施機関 |
| `prefecture` | 対象地域（`全国` または都道府県名） |
| `employeeMax` / `employeeMin` / `employeeLimitText` | 従業員数の上限・下限・原文 |
| `maxAmount` | 補助上限額（円。0は未設定） |
| `acceptanceStart` / `acceptanceEnd` | 募集開始・終了日時（ISO8601） |
| `purposes` | タイトルから推定した目的タグ |
| `url` | jGrants詳細ページURL |

## 6. マッチングロジック

### 6.1 ルールベース（evaluate）

`SUBSIDIES` の各 `match(company)` を実行し、`eligible === true` のものを採用。各制度ごとに「あてはまる点（reasons）」「確認が必要な条件（cautions）」を返す。

### 6.2 公式補助金（filterJgrants）

**絞り込み条件**（すべて満たすもののみ表示）

1. 地域: `全国` もしくは 所在都道府県と一致
2. 従業員数: `employeeMax` 以下かつ `employeeMin` 以上
3. 募集期間: 締切（`acceptanceEnd`）が未来
4. 目的: ユーザーが目的を選択し、かつ制度側に目的タグがある場合は一致を要求

**関連度ランキング**（上位ほど関連が高い）

1. 選択した目的との一致数が多い
2. 地域密着（自治体）の制度を `全国` より優先
3. 締切が近い順

表示は上位40件（超過分は件数のみ表示）。

## 7. 公式データの自動連携

- `scripts/fetch-jgrants.js` が jGrants 公開API（`GET /exp/v1/public/subsidies`）から **募集中（acceptance=1）** の補助金を取得。
- 複数キーワード（補助・助成・支援金・給付金・事業）で取得し `id` で重複排除。
- 取得結果を正規化し `data/subsidies.json` と `data/subsidies.js` を生成。
- **CORS制約**: jGrants APIはCORSヘッダーを返さないため、ブラウザからの直接呼び出しは不可。そのため「取得スクリプト → 静的データ生成 → ブラウザが同梱JSを読込」という構成にしている。
- **自動更新**: `update-subsidies.yml` が毎日APIを再取得し、差分があれば自動コミット。

## 8. 制約・免責

- 判定は**申請の目安**。要件・支給額・募集状況は簡略化している。
- 厚労省の雇用関係助成金はjGrantsに含まれないため `data.js` で補完。
- 実際の支給可否は公式の最新情報での確認が必要（画面に出典・更新日を明示）。
